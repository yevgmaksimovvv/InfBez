# Проект готов к использованию!

## Реализованный функционал:

### ✅ Криптографические алгоритмы (реализация по ГОСТ):

1. **Кузнечик (ГОСТ Р 34.12-2018)**
   - Полная реализация всех функций (mul_gf2_px, linear, x, s, s_inv, r, r_inv, l, l_inv)
   - Генерация ключей (256 бит + 10 итерационных ключей)
   - Шифрование/расшифрование блоков 128 бит
   - Файл: `algorithms/kuznechik/kuznechik.py`

2. **Стрибог-512 (ГОСТ 34.11-2018)**
   - Полная реализация хеширования
   - Все константы из ГОСТ (A, P, T, C)
   - Функции lps, gn, streebog_512
   - Файл: `algorithms/streebog/streebog.py`

3. **RSA-32768**
   - Генерация простых чисел через GMP (gmpy2)
   - Генерация ключей 32768 бит
   - Шифрование/расшифрование с padding
   - Файл: `algorithms/rsa_32768.py`

### ✅ Backend (FastAPI):

- Тройная аутентификация:
  - Фактор 1: Пароль (хеш Стрибог-512)
  - Фактор 2: OAuth (Google/Yandex) - заготовка
  - Фактор 3: Email/OTP
- Роли пользователей: Гость, Пользователь, Админ
- API для шифрования (RSA/Kuznechik)
- API для хеширования (Стрибог)
- Экспорт в PDF с ЭЦП
- PostgreSQL интеграция

### ✅ Frontend (React):

- Красивый современный дизайн с Tailwind CSS
- Страницы: Login, Dashboard, Crypto, Documents
- Интеграция с API
- Контекст аутентификации

### ✅ Инфраструктура:

- Docker Compose для PostgreSQL
- Alembic для миграций
- Структура проекта готова

## Следующие шаги для запуска:

1. Установите зависимости (см. INSTALLATION.md)
2. Настройте файл `.env` для бэкенда (на основе `.env.example`)
3. Запустите бэкенд и фронтенд (таблицы создадутся автоматически при первом запуске)
5. Настройте OAuth провайдеры (Google/Yandex) в `.env`
6. Настройте SMTP для отправки OTP кодов
7. Создайте первого пользователя через API или напрямую в БД

## Важные замечания:

- Алгоритмы реализованы строго по ГОСТ
- Все константы соответствуют стандартам ГОСТ
- RSA использует GMP для длинной арифметики
- Стрибог реализован в режиме 512 бит
